# 基于订阅的跨链智能合约访问机制


## 项目组织介绍

`chain         底层区块链目录`

`evaluations   实验评估目录`

`gateway       预言网关及对比方案开发文件夹（包含源码） `

`test          开发过程中测试文件`

## 评估实验介绍

### 实验一

* 目标：评估框架的可行性，以及部分性能指标。
* 设置：
  * ETH*2
  * Fisco*1
  * oraclegate*3
  * CCSCComponet*1
* 任务负载：
  1. 1-100个合约状态更新一次。
  2. 一个合约更新10-1000次。
  3. 1-100合约访问一次。
  4. 一个合约访问10-1000次。


### 实验二
* 目标：对比几种互操作性模式，比较其性能指标。
* 设置：
    * ETH*2
    * Fisco*1
    * oraclegate*50
    * CCSCComponet*1
* 任务负载：
    1. 1-100个合约状态更新一次。
    2. 一个合约更新10-1000次。
    3. 1-100合约访问一次。
    4. 一个合约访问10-1000次。
